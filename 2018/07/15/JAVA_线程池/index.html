线程池

Posted on 2018-07-15 In JAVA

什么是线程池

线程池是一套完整的线程管理机制。

为什么要用线程池

在一些特定的情况下，需要频繁的开启线程去执行任务，而且任务的执行时间又很短。那么线程的创建和销毁就会频繁的发生，对资源的消耗会比较大。这时候就需要使用线程池，对线程进行复用，降低了线程的创建和销毁次数，从而降低系统的资源消耗

怎么使用

一般使用的话，可以直接通过 Executors 类，快捷实现不同类型的线程池。

  1. Executors.newCachedThreadPool()：无限线程池。
  2. Executors.newFixedThreadPool(nThreads)：创建固定大小的线程池。
  3. Executors.newSingleThreadExecutor()：创建单个线程的线程池。

threadPool.execute(new Job());

线程池调用 execute 就可以提交任务，并根据情况去开始任务或者对任务做排队处理等

线程池的核心是 ThreadPoolExecutor 类，
Executors 的快捷生成也是通过对 ThreadPoolExecutor 类的操作完成的

ThreadPoolExecutor(int corePoolSize,
                          int maximumPoolSize,
                          long keepAliveTime,
                          TimeUnit unit,
                          BlockingQueue<Runnable> workQueue)
参数后面再详细讲解

线程池的原理

线程池的运作流程

把线程池比喻为一个加工厂。
线程就是加工厂的员工，一个员工同一时间只能处理一个加工原件。
加工厂还有一个仓库，当所有员工都在工作的时候，新到的加工原件就会运到仓库里。
仓库是有容量限制的，最多存储 m 个加工原件。
加工厂老板最多只会请 n 个员工，所以说加工厂的员工数量是有一个限制的。

初始状态：加工厂仓库配置好了，但是一个工人都没有。

当第一个加工原件到来的时候，工厂没有工人，就会去招一个工人来干活。

后面继续有加工原件到来的时候，就会视情况而定了。

[1.1] 如果有空闲的工人，就立即分配给空闲的工人去加工。
[1.2] 如果没有空闲的工人，那就看看工厂的员工数量是否达到 n 个。
    [1.2.1] 如果没有，则继续招人，并分配任务。
    [1.2.2] 如果到达了，则需要看看仓库是否已满。
        [1.2.2.1] 如果未满，则将加工原件放在仓库里，排队等待员工。
        [1.2.2.2] 如果已满，则加工厂会拒绝接受这个加工原件。

线程池内的数量控制

继续上面的比喻。

员工分为两种，核心员工和非核心员工。
分别对应核心线程和非核心线程。

区别：
    核心线程，默认情况下不会因为超时，而被回收，除非特意设置了 allowsCoreThreadTimeOut。
    非核心线程，在超时后，会被销毁回收。

类比一下就是，核心员工进来后，正常情况下，不用担心被辞退。
非核员工，当一段时候没活干之后，就会被辞退了。

那么工厂的员工数量如何控制？

首先加工厂会先做好人员配置，即需要事先想好，最多招多少个核心员工以及多少个非核心员工。

代码里对应的就是在新建线程池的时候，传入的2个参数决定的。

int corePoolSize,int maximumPoolSize
corePoolSize 代表核心员工的数量
maximumPoolSize 代表员工数量最多多少个

非核心员工的数量就可以理解为 maximumPoolSize - corePoolSize，如果maximumPoolSize 小于等于 corePoolSize ，那么 maximumPoolSize 就没有意义，也就代表不会有非核心员工。

如果 maximumPoolSize > corePoolSize，代表可以有非核心员工。那么上面的流程其实是不准确的。在 [1.2] 的时候，用作判断的 n 其实是 corePoolSize。 maximumPoolSize 会在 [1.2.2.2] 之后还会做一次判断。改动后的流程如下：

[1.1] 如果有空闲的工人，就立即分配给空闲的工人去加工。
[1.2] 如果没有空闲的工人，那就看看工厂的员工数量是否达到 corePoolSize。
    [1.2.1] 如果没有，则继续招聘核心员工，并分配任务。
    [1.2.2] 如果到达了，则需要看看仓库是否已满。
        [1.2.2.1] 如果未满，则将加工原件放在仓库里，排队等待核心员工。
        [1.2.2.2] 如果已满，则判断当前员工总数是否达到 maximumPoolSize。
            [1.2.2.2.1] 如果没有，则招聘非核心员工，分配任务。
            [1.2.2.2.2] 如果达到，加工厂会拒绝接受这个加工原件

参数解析

int corePoolSize,int maximumPoolSize 上面已经讲解了

接下来

long keepAliveTime, TimeUnit unit 
keepAliveTime：时间的值
unit：时间单位

他们是一体的，代表着超时时间。
就是非核心线程在空闲后，多久会被回收。
类比过来就是，非核心员工在多久没活干之后，就会被工厂辞退的意思。

最后 

BlockingQueue<Runnable> workQueue
workQueue 就是仓库的类比，是一个阻塞队列，分为有界队列和无界队列。
线程超时关闭的功能就是依赖于此队列实现的。
核心线程和非核心线程在完成任务后，都会去从队列里查找是否有新的任务，但查找方式有区别。
非核心线程，是通过阻塞队列的 poll 方法，而核心线程是通过 take 方法。
这样就能理解了为什么核心线程不会超时，而非核心线程会超时了。
